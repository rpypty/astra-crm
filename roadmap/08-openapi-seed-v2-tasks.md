# Новые задачи: интеграционный шаг 1 и 2

## Задача 1. OpenAPI codegen / контрактная чистка (доп. этап INT-001)

### Цель
Устранить ручные предположения в frontend API-клиенте и заставить frontend быть привязанным к OpenAPI-спеку без drift.

### Что делаем
- Подготовить скрипт генерации типов и клиента из `docs/openapi.yaml`.
- Настроить стабильный CI-safe pipeline для обновления сгенерированных артефактов:
  - отдельная npm/yarn цель для генерации;
  - проверка, что `git diff` после генерации пустой в CI.
- Перевести frontend API layer на сгенерированный контракт (минимально инвазивно):
  - endpoint-методы вместо ручных интерфейсов/типов;
  - строгая типизация request/response/ошибок.
- Добавить защиту от drift:
  - тест/проверка на валидность `docs/openapi.yaml`;
- Пройтись по текущим фрагментам `src/lib/api.ts` и API hooks и убрать ручные дублирующие типы.

### Критерии готовности
- Удалены/заменены ручные типы, которые дублируют OpenAPI-модели.
- Код сборки frontend проходит без ручных кастомных casting-обходов.
- При изменении спеки контрактный код пересобирается и типовая ошибка видна на этапе build/test.

## Задача 2. Seed Data v2

### Цель
Добавить релевантные локальные демо-данные, чтобы сразу был «живой» дашборд и сценарии с репутационно полным потоком.

### Что делаем
- Расширить seed данные для:
  - активной смены с несколькими requisites;
  - импортов inbound/outbound;
  - payout transfers и reconciliation runs;
  - audit-событий по ключевым операциям;
  - mismatch/acceptance кейсов для dashboard.
- Добавить локальный seed с закрытой сменой и с расхождением для проверки `closed_with_discrepancy`.
- Обновить seed scripts/fixtures и `docker-compose`/README, чтобы данные применялись одним запуском.
- Отдельными сценариями обеспечить:
  - демонстрационный teamlead dashboard с заполненными графиками;
  - готовые кейсы для e2e-интеграции.

### Критерии готовности
- В `docker-compose up` после старта доступен минимальный воспроизводимый набор сценариев.
- UI показывает статистику/графики без ручного ввода базовых данных.
- Покрыты smoke-пути:
  - take requisite → turnovers → payouts → imports → close shift.
  - mismatch + accept mismatch + close shift with discrepancy.

### Артефакты
- `astra-crm-backend/internal/...` seed scripts/fixtures
- `docs/open-questions.md` (если появляются новые бизнес-оговорки)
- `docker-compose.yml`, `.env.example`, `README` при необходимости
